Nom du Projet : Pipeline de Surveillance des Transactions Financières en Temps Réel
Auteur : Mohamed Sakhri
Date de Début : 22/12/2025
Date de Fin : 22/12/2025

Le scénario : Des transactions bancaires affluent en continu (simulées).
              Nous devons les ingérer, détecter celles qui sont potentiellement frauduleuses (basé sur des règles simples ou un modèle), les stocker, et permettre à des analystes de les requêter via SQL immédiatement.



Implémentation étape par étape :
Étape 1 : Infrastructure (Docker)
Étape 2 : Ingestion des données (Kafka)
Étape 3 : Traitement des données (Spark)
Étape 4 : Stockage des données (HDFS)
Étape 5 : Analyse des données (Impala)
Étape 6 : Visualisation des données (Tableau de bord)
